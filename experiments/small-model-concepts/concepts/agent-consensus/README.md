# Блок 2. Agent Consensus / Рой агентов

Эти концепции проверяют не "толпу болтающих моделей", а строгий консилиум с ролями и ограничениями.

## `strict-skill`

Один skill получает факты и отвечает шаблоном. Модель не участвует.

## `verify`

Один ответчик формирует ответ из фактов, второй verifier проверяет:

- ответ содержит только значения из facts;
- сущность выбрана правильно;
- нет ложных предпосылок.

## `council`

Три роли:

- `router` - определяет слой и intent;
- `retriever` - получает MCP/API facts;
- `critic` - проверяет ответ и ловушки.

Гипотеза: council нужен только на неоднозначных и adversarial вопросах.
